# 运维流程计划

本文档定义运维工作的标准端到端流程。

## 1. 目标与范围

- 规范日常运维、事故响应、发布后观察与周期性报告。
- 确保关键动作可追踪、可复盘、可持续改进。
- 范围：本仓库生产站点及其 CI/CD 工作流。

## 2. 角色与职责

- 值班工程师：分诊告警、缓解事故、维护事件时间线。
- 发布负责人：执行发布清单并协调发布后观察。
- QA/维护者：验证冒烟检查并跟踪 flaky 测试。
- 技术负责人：复核事故经验，审批高影响行动项。

## 3. 流程分阶段

### 阶段 A：日常运维

1. 巡检监控与告警渠道。
2. 按严重级别与用户影响分诊未结事件。
3. 更新状态与责任人。
4. 产出交接说明。

参考文档：

- [值班响应手册](oncall-playbook.zh.md)
- [值班交接模板](oncall-handover-template.zh.md)

### 阶段 B：事故处理

1. 发现并确认。
2. 快速止损（优先恢复可用性）。
3. 定位根因。
4. 恢复并验证。
5. 记录完整事件时间线。

参考文档：

- [事故复盘模板](postmortem-template.zh.md)

### 阶段 C：发布后跟踪

1. 确认发布完成与部署信号。
2. 执行发布后 24 小时观察清单。
3. 记录回归问题与回滚决策。

参考文档：

- [发布运行手册](../release/release-runbook.zh.md)
- [发布后 24 小时观察清单](post-release-24h-checklist.zh.md)

### 阶段 D：周报与月报

1. 输出每周运营周报。
2. 汇总月度稳定性指标与行动项。
3. 评审重复风险并排定下月改进优先级。

参考文档：

- [每周运营周报模板](weekly-ops-report-template.zh.md)
- [月度稳定性报告模板](monthly-stability-report-template.zh.md)

## 4. 升级规则

- SEV-1：立即升级至技术负责人和仓库 Owner。
- SEV-2：影响超过 30 分钟或影响核心路径时升级。
- SEV-3/4：默认纳入常规待办，趋势恶化时再升级。

## 5. 每周期完成标准

仅当以下条件全部满足，才视为一个周期完成：

- 事故具备记录、责任人和结果。
- 发布后检查已执行并留档。
- 周报或月报按时发布。
- 未结行动项具备截止日期与责任人。

## 6. 评审节奏

- 每周：运维同步会，回顾值班摘要。
- 每月：稳定性评审，复盘 KPI 趋势与风险。
- 每季度：流程审计与机制优化。
