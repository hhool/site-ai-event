# 发布后 24 小时观察清单

每次生产发布后，请在前 24 小时按本清单执行巡检。

## 发布信息

- 版本/标签：
- 发布时间（UTC）：
- 发布负责人：
- 风险等级：Low / Medium / High

## 0-1 小时检查

- [ ] 首页可访问（`/en`、`/zh`）
- [ ] 至少一个详情页可访问
- [ ] 根路由重定向（`/` -> `/en`）正常
- [ ] 核心外链可打开（demo/github/官网）
- [ ] 线上冒烟测试通过（`pnpm test:e2e:prod`）

## 1-6 小时检查

- [ ] 错误率相对基线稳定
- [ ] 未触发 SEV-1/SEV-2 事件
- [ ] CI/发布工作流执行成功
- [ ] 失败检查项无异常增长
- [ ] 无紧急用户侧投诉

## 6-24 小时检查

- [ ] 可用性达到目标
- [ ] 无未解决的高优先级回归
- [ ] 告警状态正常或已确认
- [ ] 轻微问题已记录行动项
- [ ] 周报已同步本次发布结果

## 回滚判定门槛

若满足任一条件，应触发回滚或热修：

- [ ] 核心链路可复现故障
- [ ] 错误率持续高于阈值
- [ ] 发布自动化故障影响生产交付
- [ ] 发布后发现安全风险

## 沟通记录

- 首次发布通知时间：
- 1h 状态同步时间：
- 6h 状态同步时间：
- 24h 收口同步时间：

## 收口确认

- [ ] 24h 观察完成
- [ ] 发布标记为稳定
- [ ] 后续行动项已分配
- [ ] 是否需要事故复盘：Yes / No
