# Branch Protection Guide

This document defines the recommended branch protection settings for `main`.

## Scope

- Repository: `hhool/site-ai-event`
- Branch: `main`
- Goal: block low-quality or unsafe changes from merging

## Prerequisites

Before enabling strict checks, ensure these workflows have run at least once on `main` so GitHub can list their status checks:

- `.github/workflows/ci.yml`
- `.github/workflows/codeql.yml`
- `.github/workflows/commitlint.yml`
- `.github/workflows/pr-body-check.yml`

## Recommended Rule (GitHub UI)

Go to:

- `Settings` -> `Branches` -> `Add branch protection rule`
- Branch name pattern: `main`

Enable the following options:

1. `Require a pull request before merging`
2. `Require approvals`: `1`
3. `Dismiss stale pull request approvals when new commits are pushed`
4. `Require conversation resolution before merging`
5. `Require status checks to pass before merging`
6. `Require branches to be up to date before merging`
7. `Do not allow bypassing the above settings`
8. `Restrict pushes that create files larger than 100 MB` (if available in your plan)

## Required Status Checks

Select these checks from the status-check list (names may vary slightly by UI):

- `quality / quality`
- `Analyze / Analyze (javascript-typescript)`
- `commitlint / commitlint`
- `validate-pr-description / validate-pr-description`

If the exact labels differ, select the checks generated by these workflows instead:

- `.github/workflows/ci.yml`
- `.github/workflows/codeql.yml`
- `.github/workflows/commitlint.yml`
- `.github/workflows/pr-body-check.yml`

## Optional Hardening

- Enable `Require signed commits` if your team already uses signed commits.
- Enable `Require linear history` if you want to disallow merge commits.
- Restrict force pushes and branch deletions for all non-admin users.

## Merge Strategy Recommendation

- Keep `Squash merge` enabled.
- Disable `Merge commit` for a cleaner history.
- Keep commit titles meaningful and Conventional Commit-compatible.

## Rollout Checklist

1. Verify all required checks are green on a recent PR.
2. Enable the branch protection rule on `main`.
3. Open a test PR and verify merge is blocked until all checks pass.
4. Confirm direct push to `main` is blocked for non-admin users.
